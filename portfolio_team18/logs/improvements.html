Overall Project Comments
  • Sairam Krishnan (sbkrishn) - Great project overall. A few minor suggestions:
    • Don't force students to write about what they liked or didn't like about this project.
    • Recommend Gliffy for creating Sequence Diagrams and Statecharts, rather than Dia.
    • Warn students about merge issues in GitHub (in other words, how important it is to pull and update repo before pushing and to coordinate pushes.
  • Yi Huo (yhuo) - Overall, I liked the project.
    Possible improvements:
      Warn students in advance that github does not work well with excel files.
  • Yigit Bilgen (mbilgen) - Overall, I really liked the project.
    Possible improvements:
    • Add simulation time info on the .stats files, along with the stdout input.
    • Typing the same message periods, translator names, their arguments and CAN IDs over and over was very tedious. It would be great if there was a way to define certain combinations in .mf files. i.e:

      #define DoorClosedFrontLeft DOOR_CONTROL_PERIOD DOOR_CLOSED_[FRONT][LEFT]_CAN_ID DoorClosedCanPayloadTranslator Front Left
      ...
      0.50s I DoorClosedFrontLeft = true
      0.50s I DoorClosedFrontRight = true

      etc.

  • Abhijit Lele (alele) - Mostly a good project. Some improvements that could be made:
    • For one of the earlier projects, make students read the Lessons Learned from previous year and submit a text file with the information that they think is useful. They're incredibly useful.
    • Provide more templates/sample files to get started on the Controllers and CanPayloadTranslators. At the very least, offer more suggestions on what kind of things we might need to add.
