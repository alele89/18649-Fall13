Controller Files

  • CarButtonControl - Controller that handles car button presses (buttons within the car).
  • CarPositionControl - Controller that tracks the car position and informs the user what floor the car is or was at.
  • Dispatcher - Controller that determines the next floor, hallway, and direction that the car must travel to.
  • DoorControl - Controller corresponding to a single door facing a particular hallway; handles opening, closing, and reversal.
  • DriveControl - Controller that handles the car's drive and thereby controls its motion.
    NOTE: This controller runs with a period of 5 ms, instead of the default (10 ms). The reason for this is because after detecting that the commit point is reached, the elevator still travels at FAST speed before slowing down (10 ms * 5 m/s = extra 5 cm). This is because the state chart does not allow actions on transitions. By reducing the period, we hope to reduce the overshoot by 2.5 cm. This should reduce the slack needed on commit point calculations, and allow the elevator to move at FAST speed for a larger amount of time, increasing performance.
    Approved by: Gunjan.
  • HallButtonControl - Controller that handles hall button presses (the up and down buttons in the hall, outside the car).
  • LanternControl - Controller that manages the car lanterns, which tell the passenger what direction the car is going in.

Translator Files

  • DesiredDwellCanPayloadTranslator - Handle the mDesiredDwell network message, which specifies how long the doors need to remain open.
  • DesiredFloorCanPayloadTranslator - Handle the mDesiredFloor network message, which specifies the next floor that the dispatcher wants the car to stop at.
  • DriveCommandCanPayloadTranslator - Handle the mDrive network message.
  • DoorMotorCanPayloadTranslator - Handle the mDoorMotor network message.
  • DriveSpeedCanPayloadTranslator - Handle the mDriveSpeed network message.

Utility Files

  • Utility - Contains utility functions to handle some repeatable configuration and information retrieval tasks.
  • MessageDictionary - Contains the list of possible network messages and their corresponding can ids

Runtime Monitors

  • Proj7RuntimeMonitor - Monitors various performance statistics.
  • Proj11RuntimeMonitor - Validates the elevator against high level requirements R.T-6 ~ R.T-10.
